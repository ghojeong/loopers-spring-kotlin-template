# Weekly I Learned 5 (Read Optimization)

SBS 가요대전 방청권 티케팅 서비스를 맡게 됨.
인기에 대비해 RPS 1.7만을 버틸 수 있도록 하라는 요구사항이 들어왔고, Redis 캐시에 대한 교육 없이는 업무를 절대 못해냈을 거임.

중복 티케팅을 할 수 없도록 휴대폰 본인인증을 통해 중복된 인물인지 검사해야했는데, Redis setnx 연산을 사용해 빠르고 원자적인 중복검사가 가능했음.
뿐만 아니라 Redis 에 미리 써두고 나중에 DB 에 Write Back 하는 전략을 취해서 DB 의 쓰기로 인한 트랜잭션 부하를 줄일 수 있었음
